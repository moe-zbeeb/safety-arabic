## Project Purpose

This repository is an Arabic LLM safety research project. It contains Arabic safety fine-tuning data, supervised fine-tuning scripts, DPO scripts, AraSafe/AraTrust evaluation scripts, Arabic boundary-set robustness experiments, safety-direction activation experiments, and archived experiment results.

## Main Training Workflow

Prefer:

```bash
python Train/train_flexible.py \
  --model_path /home/zbibm/Safety-Arabic/models/Fanar-1-9B \
  --dataset_path '/home/zbibm/Safety-Arabic/Training Corpus/mix70-30.jsonl' \
  --output_dir /home/zbibm/Safety-Arabic/output/fanar_mix70_30 \
  --num_epochs 1 \
  --batch_size 16 \
  --gradient_accumulation_steps 2 \
  --learning_rate 1e-5 \
  --save_steps 50
```

`Train/train.py` is hard-coded for one historical setup. Use it only when intentionally reproducing that setup or when a runner patches it.

## Known Issues And Risks

`benchmarking-scripts/benchmark_safety.py` has a fallback bug in `evaluate_with_guard`: if Qwen3Guard output is not parseable, it checks the stale loop variable `response` instead of the response corresponding to the current output. Use the indexed pattern from `benchmarking-scripts/benchmark_curriculum.py` or the parse-failure approach from `DPO/Eval/eval.py`.

`benchmarking-scripts/benchmark_all_combinations.sh` calls `Train/train_flexible.py` with old argument names:

```bash
--model --dataset --output
```

The actual script expects:

```bash
--model_path --dataset_path --output_dir
```

`Train/run_curriculum_learning.sh` has a dataset path mismatch as noted above.

Many scripts contain hard-coded absolute paths. Always inspect paths before running on a new machine.

Some scripts delete output directories or checkpoints. Inspect any `rm -rf` or `shutil.rmtree` path before running.

`boundary_set/translate_msa_to_arabizi.py` is a Colab-exported script and contains placeholder API-key logic. Do not add real keys to it.
